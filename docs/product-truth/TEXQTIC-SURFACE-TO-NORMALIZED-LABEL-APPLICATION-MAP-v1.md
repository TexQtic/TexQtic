## 6. Temporary Continuity-Shorthand vs Normalization-Now Rules

- A current label may remain temporarily only when it is a continuity aid and a neighboring badge,
  sublabel, or heading split makes the canonical family, overlay, plan, or downstream-ownership
  boundary explicit.
- Composite headings that fuse family, plan, and overlay truth must be normalized now at planning
  level; later implementation-facing work should split them rather than preserve them as one title.
- Aggregator module labels that imply network governance, automated routing, negotiation ownership,
  execution ownership, or finance ownership must be normalized now at planning level; they are not
  safe temporary shorthand.
- WL admin wording may remain temporarily only where the overlay-owned nature is explicit and the
  label does not stand in for the whole tenant-admin architecture.
- Discovery labels tied to supplier, manufacturer, company, catalog, or profile language may remain
  temporarily only if they are explicitly framed as discovery-safe and non-owning.

## 7. Allowed and Disallowed Application Interpretations

Allowed:

- headings may split one current surface into separate family, plan, and overlay segments rather
  than forcing one fused label to carry all meanings
- Aggregator workspace wording may remain when it is paired with curated-directory and
  intent-handoff framing rather than exchange-office framing
- downstream destinations reached from Aggregator context may remain visible if they are labeled as
  shared or downstream continuation
- control-plane tower labels may remain because they are already platform-supervision language
- `Tenant plan metadata` or similar phrasing may remain as package visibility language without
  claiming full billing or entitlement completeness

Disallowed:

- any application of WL labels that reads as a peer family or whole tenant-admin substitute
- any application of Aggregator labels that reads as a peer family, separate full office, or owner
  of downstream execution, negotiation, finance, or taxonomy administration
- any application of plan/package labels that reads as family identity or runtime-mode truth
- any application of downstream labels reached from Aggregator context that hides the downstream or
  shared ownership boundary
- any application of supplier/manufacturer/profile labels in Aggregator context that implies full
  B2B taxonomy ownership transfer

## 8. Blockers and Edge Conditions Inside The Map Boundary

- descriptive dashboard artifacts still self-describe as canonical or authoritative in places, so
  later implementation-facing work must normalize current labels without restoring those older docs
  as stronger authority than the five controlling artifacts and this map
- current shell names are code-facing continuity anchors, so the smallest later action for many
  rows is a visible label split, badge, or sublabel rather than a code identifier rename
- White Label Co remains a preserved Layer 0 hold, but it does not materially block this bounded
  mapping pass
