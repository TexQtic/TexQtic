## Why The Other Surfaces Are Not Controlling

### Current Layer 0 support surfaces are not the sequencing winner

- `OPEN-SET.md`, `BLOCKED.md`, `SNAPSHOT.md`, and `DOCTRINE.md` are live and controlling for
  posture, hold state, restore context, and doctrine rules.
- None of them independently issues the current exact repo-level next-step pointer.

### The opening-layer sequencing file is no longer controlling

- It still carries the stale instruction that the current lawful next move is one same-hold
  design-clarification unit.
- The chronology audit proves that clarification, verdict, and closure-recorded consumption of
  that same-hold path already happened on 2026-04-14.
- It therefore remains a referenced live-era surface but no longer survives as current controlling
  next-step authority.

### The `-v1` trilogy is no longer controlling

- Each `-v1` file self-classifies as history-only baseline and says it is no longer the active
  sequencing or roadmap base.
- Current top-of-stack reconciliation respects those self-demotions.
- Current Layer 0 references to the `-v1` trilogy are inherited pointer residue, not surviving
  proof of current control.

### The `-v2` stack is no longer controlling

- Current Layer 0 explicitly lists the `-v2` trio as historical reconciliation inputs.
- The `-v2` next-delivery plan and gap register still contain live/current self-labeling, but that
  self-promotion loses to current Layer 0 precedence and the post-GOV-OS chronology.
- The `-v2` roadmap also says it is not the live sequencing authority.

### Descendant family anchors are not controlling

- Aggregator, Subscription, B2C, and White Label family docs are mature and useful.
- None of them is a top-of-stack repo-level selector.
- Their maturity is precisely the overread risk this pass is designed to contain.

1. Subscription maturity can be mistaken for current-next authority because its chain is deep and

These overread risks do not alter the reconciled live spine.
