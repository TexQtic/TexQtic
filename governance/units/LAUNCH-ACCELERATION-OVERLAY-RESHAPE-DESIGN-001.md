### B. Artifact Design — Exact Fields

#### 1. Launch Critical Path Register

**Purpose**

Maintain one bounded current-state view of launch-critical serial and parallel dependencies so TexQtic can see which launch-relevant family or slice is actually holding up faster lawful next-opening selection.

**Required fields**

| Field | Allowed form | Meaning |
| --- | --- | --- |
| `entry_id` | stable short ID | Unique row identifier for the register entry |
| `subject` | exact family, child candidate, or support slice name | The launch-relevant thing being tracked |
| `subject_kind` | `FAMILY` \| `CHILD_CANDIDATE` \| `SUPPORT_SLICE` | Prevents broad family and exact child rows from being conflated |
| `source_authority` | path or explicit authority list | The current repo-truth source the row is derived from |
| `current_posture` | current authority posture string | Current state as preserved by controlling repo truth |
| `critical_path_role` | `CRITICAL` \| `SUPPORTING` \| `LATER` | Whether the row is currently on the launch-critical path |
| `blocking_dependency` | exact blocker or `none` | The immediate dependency holding movement back |
| `next_bounded_move` | one bounded move statement | The minimum lawful next move implied by current truth |
| `execution_relation` | `SERIAL` \| `PARALLEL` | Whether the row must move in sequence or may move alongside another bounded row |
| `dependency_unlock_value` | `LOW` \| `MEDIUM` \| `HIGH` | How much future movement this row unlocks if resolved |
| `last_refreshed` | `YYYY-MM-DD` | Date the row was last refreshed from controlling truth |

**Optional fields**

| Field | Allowed form | Meaning |
| --- | --- | --- |
| `parallel_group` | short group label | Names a bounded parallel cluster when `execution_relation` is `PARALLEL` |
| `unlock_scope` | short text | Names what later movement becomes easier or lawful if this row is resolved |
| `assumption_note` | short text | Records one assumption affecting the row |
| `risk_note` | short text | Records one material drag or uncertainty on the row |
| `change_note` | short text | Records what changed since the prior refresh |

**Update trigger**

- refresh after any product-facing close
- refresh after any material planning-truth change that changes launch dependency shape, current contender posture, or launch-critical ordering
- otherwise remain unchanged

**Owner / authority relationship**

- owned as visibility-only overlay content
- derived from Layer 0 plus live product-truth authorities only
- never overrides those authorities

**Serial vs parallel representation**

- `execution_relation: SERIAL` means the row is currently sequence-bound
- `execution_relation: PARALLEL` means the row may move with another bounded row
- `parallel_group` may appear only when `execution_relation: PARALLEL`

**Dependency-unlock representation**

- `dependency_unlock_value` expresses unlock force as `LOW`, `MEDIUM`, or `HIGH`
- `unlock_scope` may name exactly what later movement is unlocked

**Anti-sprawl rule**

- this register may contain only current launch-critical or direct blocker rows
- no waves, no backlog inventory, no long-horizon roadmap content
- maximum active row count: 7

Not this: not a roadmap, not a wave plan, not a backlog table, and not a full family catalog.

#### 2. Next-Opening Shortlist Matrix

**Purpose**

Keep a bounded comparison of the best current contenders for the next lawful opening decision so operator judgment can happen faster without turning the shortlist into opening authority.

**Exact contender limit**

- minimum: `0` when no contender is exact enough to shortlist
- maximum: `5` contenders at one time

**Required fields**

| Field | Allowed form | Meaning |
| --- | --- | --- |
| `contender_id` | stable short ID | Unique row identifier |
| `candidate_name` | exact candidate or child name | The contender being compared |
| `source_authority` | path or explicit authority list | The source the contender is derived from |
| `boundedness_verdict` | `PASS` \| `NEEDS_REDUCTION` \| `NOT_LAWFUL` | Whether the contender is exact enough for shortlist use |
| `launch_impact_score` | `1` to `5` | Higher means stronger launch effect |
| `dependency_unlock_score` | `1` to `5` | Higher means stronger unlock effect on later movement |
| `execution_size` | `XS` \| `S` \| `M` \| `L` \| `XL` | Smaller means faster bounded execution |
| `design_clarity_score` | `1` to `5` | Higher means clearer current target state |
| `reversibility_score` | `1` to `5` | Higher means easier rollback or limited blast radius |
| `cost_of_delay_score` | `1` to `5` | Higher means greater penalty for leaving the contender unopened |
| `recommended_rank` | `1` to `5` or `none` | Ranking output for contenders with `boundedness_verdict: PASS` only |
| `not_opening_reason` | short text | Why the contender is not already open |

**Exact ranking dimensions**

- `launch_impact_score`
- `dependency_unlock_score`
- `execution_size`
- `design_clarity_score`
- `reversibility_score`
- `cost_of_delay_score`

**Boundedness evaluation rule**

`boundedness_verdict: PASS` is lawful only when all of the following are true:

1. the exact problem target is named
2. exact in-scope and out-of-scope boundaries are already clear
3. no unresolved design-gate or known hidden dependency blocker is being ignored
4. the contender can be discussed without widening back into broad family shape

If the contender is real but still lane-mixed or structurally broad, use `NEEDS_REDUCTION`.
If current truth already shows it cannot lawfully open, use `NOT_LAWFUL`.

**Ranking rule**

- only `PASS` contenders receive `recommended_rank`
- tie-break order is:
  1. higher `launch_impact_score`
  2. higher `dependency_unlock_score`
  3. smaller `execution_size`
  4. higher `design_clarity_score`

**Authority relationship**

- the matrix informs next-opening decision work only
- it does not authorize an opening
- a contender appearing at rank 1 is still not `OPEN`

Not this: not a whole-backlog ranking system, not candidate-truth replacement, and not an opening decision by itself.

#### 3. Rolling Launch Window Note

**Purpose**

Maintain one bounded rolling launch-range view derived from the current critical path and shortlist so launch timing can be discussed without treating the note as a commitment surface.

**Exact allowed outputs**

- one `best_case_window`
- one `likely_window`
- one `risk_adjusted_window`
- one explicit assumption set
- one refresh reason
- optional change note and confidence note only

**Required fields**

| Field | Allowed form | Meaning |
| --- | --- | --- |
| `note_date` | `YYYY-MM-DD` | Date of the note |
| `baseline_authorities` | explicit source list | The authorities used to derive the note |
| `best_case_window` | date-range string | Earliest credible launch window if no new material blocker appears |
| `likely_window` | date-range string | Central expected launch window under current assumptions |
| `risk_adjusted_window` | date-range string | Conservative launch window including known drag and likely slippage |
| `assumptions` | list of explicit assumptions | The assumptions the range depends on |
| `refresh_reason` | short text | Why the note is being refreshed now |

**Optional fields**

| Field | Allowed form | Meaning |
| --- | --- | --- |
| `confidence_note` | short text | Qualifies how stable the note currently is |
| `change_since_last_note` | short text | States what materially changed |
| `invalidated_by` | short text | Names the event that would invalidate the note |

**Best-case / likely / risk-adjusted structure**

- each window must be a range, not a single date
- single-date outputs are forbidden
- each range must remain derived from the current critical path and assumption set

**Assumption handling**

- maximum assumption count: 5
- each assumption must be explicit and reviewable
- if a range depends on an unstated assumption, the note is incomplete

**Refresh conditions**

- refresh after any product-facing close that changes launch reality
- refresh after any material planning-truth change that changes the critical path or shortlist
- refresh when the embedded stagnation rule fires
- otherwise remain unchanged

**Authority prohibition**

- this note is not commitment authority
- this note is not investor-grade promise authority
- this note is not product-scope authority

Not this: not a launch commitment, not investor guidance, not a milestone plan, and not an externally promissory date surface.
